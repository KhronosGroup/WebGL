WebGL Extension Template

Comments in this extension template are highlighted in green. This entire section is one big comment.

This template for WebGL extensions is derived from the OpenGL extension template. Refer to the OpenGL extension template for full documentation of the content that should be contained in the sections below. Because WebGL is fundamentally a Web API, its extensions are specified in HTML for easier hyperlinking.

Because most WebGL extensions are expected to simply mirror existing OpenGL and OpenGL ES extensions, it is desirable to keep the WebGL extension specifications as small as possible and simply refer to the other specifications for the behavioral definitions.

Name

The name of the extension. This is also the string which is passed to the WebGLRenderingContext getExtension API to enable the extension. For example, if the extension name were OES_texture_float, then the extension would be fetched and enabled with a call to getExtension("OES_texture_float").

Per WebGL convention, the GL_ prefix is dropped from all extension and enum names, and the gl prefix dropped from, and capitalization adjusted of, all function names.

Contact

The email address of the contact of the specification. Prefer to use 'at' instead of the @ character to reduce spam.

Contributors

Names and email addresses of individuals who contributed to the development of the specification.

Version

Last modified date and revision number of this specification. Example:

Last modified date: November 29, 2010
Revision: 1

Number

The number of the extension in the WebGL extension registry.

Dependencies

The WebGL specification version against which this extension is written. For example:

Written against the WebGL API 1.0 specification.

Overview

A description of what this extension actually does. If this extension simply mirrors an OpenGL or OpenGL ES extension, it is recommended to simply link to that other extension's specification. For example:

This extension exposes the OES_texture_float functionality to WebGL. There are no WebGL-specific behavioral changes. Consult the above extension for all documentation, issues, and new functions and enumerants.

IDL

This section should contain a link to a file containing the Web IDL for the extension. The file should be named according to the name of the extension: for example, OES_texture_float.idl for the OES_texture_float extension. Enums and functions introduced by the extension should be specified as const GLenum and methods on the extension object. The extension IDL should be written assuming that the typedefs in webgl.idl and typedarrays.idl have already been defined, and may implicitly reference other web platform specifications.

Example contents for a hypothetical extension: 

module webgl {

interface OES_foo_bar {
  const GLenum NEW_ENUM = 0xF00;

  void doSomethingNew(GLenum state, GLuint value);
}; // interface OES_foo_bar

}; // module webgl
